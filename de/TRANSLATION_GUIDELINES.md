## German translation guidelines

- Address the learner informally and consistently: use lowercase "du", "dir", "dich", "dein", and "deine". Do not use formal "Sie" or capitalized "Du".
- Avoid literal translations and focus on readability for German readers.
- Do not try to explain concepts in your own words, if it's not in the original string.
- Keep code and variable names in English to avoid awkward translation issues with ä, ö, ü. Readers often know just enough English to understand.
- Keep Godot/GDScript keywords, identifiers, function names, values, and code examples unchanged.
- Preserve all BBCode markup and formatting exactly, including "[code]", "[b]", "[i]", line breaks, lists, and punctuation inside code.
- Use consistent technical terms throughout the course. Join Godot code symbols naturally with German words, for example "[code]for[/code]-Schleife" and "[code]Vector2[/code]-Variable".
